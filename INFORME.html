1. Introducción

Este proyecto implementa un servidor socket multicliente en Java que permite la comunicación entre múltiples clientes conectados simultáneamente. El servidor utiliza hilos (threads) para atender a cada cliente de manera independiente, posibilitando que varios usuarios interactúen con el servidor y se comuniquen entre sí a través de mensajes.

La solución también implementa un sistema robusto de logging que registra todas las operaciones en consola y en un archivo, facilitando la auditoría y depuración del sistema.

Además, el proyecto está diseñado para soportar conexiones remotas vía ZeroTier, permitiendo que máquinas en diferentes ubicaciones puedan conectarse de forma segura sin la necesidad de complicadas configuraciones de firewall o puertos.

2. Características Principales

  • ✅ Servidor Multicliente con Hilos: Cada cliente conectado es atendido por un hilo independiente
  • ✅ Asignación Automática de Nombres Únicos: El servidor asigna nombres como Usuario, Usuario1, Usuario2, etc.
  • ✅ Comunicación Inter-clientes: Los clientes pueden enviar mensajes a uno específico o a todos
  • ✅ Sistema de Logging Completo: Todos los eventos se registran en consola y en archivo servidor.log
  • ✅ Validación de Errores Robusta: El servidor valida clientes inexistentes, formatos incorrectos, etc.
  • ✅ Soporte para Conexiones Remotas: Compatible con ZeroTier para conectar máquinas remotas
  • ✅ Mantenimiento de Funcionalidades Originales: Conserva comandos RESOLVER, CONTAR, PROVINCIAS
  • ✅ Interfaz Amigable: Menú de ayuda claro en el cliente

4.2 Flujo de Comunicación

Secuencia para enviar un mensaje entre clientes:

  1. Cliente A escribe: *Usuario1 "Hola"
  2. Se envía al servidor a través del socket
  3. El hilo del Servidor identifica el comando
  4. Busca en ConcurrentHashMap al cliente "Usuario1"
  5. Si existe, obtiene su PrintWriter
  6. Envía el mensaje formateado: [Usuario -> Usuario1] Hola
  7. Registra en log: [MENSAJE] Usuario -> Usuario1: Hola
  8. Retorna confirmación al cliente A

5. Componentes Principales

5.1 Servidor.java

Responsabilidades principales:

  • Escuchar en puerto 6789 y aceptar conexiones
  • Crear un hilo por cada cliente que se conecta
  • Gestionar lista de clientes activos
  • Asignar nombres únicos a cada cliente
  • Procesar comandos recibidos
  • Gestionar mensajes inter-clientes
  • Registrar todas las operaciones en log
  • Validar errores y notificar al cliente

5.2 Clase Interna: ClienteConectado

private static class ClienteConectado { String nombre; // Nombre único asignado String ip; // Dirección IP del cliente PrintWriter salida; // Stream para enviar datos }

Esta clase representa a un cliente conectado y almacena su información junto con el canal de salida para enviarle mensajes.

5.3 Cliente.java

Responsabilidades principales:

  • Conectarse al servidor

  • Ejecutar un hilo lector asincrónico para recibir mensajes
  • Validar formato de comandos antes de enviar
  • Mostrar mensajes de otros clientes de forma clara
  • Manejar desconexiones elegantemente

6. Detalles de Implementación

6.1 Thread-Safety (Seguridad en Hilos)

El servidor usa ConcurrentHashMap para almacenar clientes, que es thread-safe sin necesidad de sincronización explícita:

private static final ConcurrentHashMap<String, ClienteConectado> clientesConectados = new ConcurrentHashMap<>();

Esto permite que múltiples hilos accedan a la lista de clientes sin riesgo de condiciones de carrera.

6.2 Asignación de Nombres Únicos

Cuando un cliente se conecta, el servidor genera un nombre único:

private static String asignarNombreUnico(String nombreBase, PrintWriter salida) { synchronized (Servidor.class) { String nombreFinal = nombreBase; int contador = 1; while (clientesConectados.containsKey(nombreFinal)) { nombreFinal = nombreBase + contador; contador++; } return nombreFinal; } }

Se utiliza synchronized para evitar que dos clientes reciban el mismo nombre simultáneamente.

6.3 Sistema de Logging

Todas las operaciones se registran con timestamp:

[16/04/2026 14:30:45] [CLIENTE] Usuario (127.0.0.1) - CONECTADO [16/04/2026 14:30:51] [LOG Usuario] >>> FECHA [16/04/2026 14:30:56] [MENSAJE] Usuario -> Usuario1: Hola

Los logs se guardan en servidor.log en la misma carpeta de ejecución.

6.4 Validación de Errores

El servidor valida:

  • Existencia del cliente destino antes de enviar mensaje
  • Formato correcto de comandos (comillas cerradas, etc.)
  • Expresiones matemáticas válidas
  • Que un cliente no se envíe mensajes a sí mismo
  • Mensajes vacíos

8. Configuración para Conexiones Remotas (ZeroTier)

8.1 ¿Por qué ZeroTier?

ZeroTier es una red virtual que simula una red local sobre Internet:
  • ✅ Sin configuración complicada de puertos
  • ✅ Encriptación automática AES-256
  • ✅ Funciona con firewalls y NAT
  • ✅ Gratuito para hasta 100 dispositivos

8.2 Pasos Rápidos

1. Descargar e instalar ZeroTier:

  • Ve a https://www.zerotier.com/download/
  • Descarga para tu SO (Windows, Linux, macOS)
  • Instala normalmente

2. Crear una red ZeroTier:

  • Ve a https://my.zerotier.com/
  • Crea una cuenta
  • Haz clic en "Create a Network"
  • Copia el Network ID (16 caracteres hex)

3. Unirse a la red en cada máquina:

# Linux/macOS sudo zerotier-cli join a0996acf12345678 # Windows: usa la interfaz gráfica # Haz clic derecho en icono ZeroTier > Join Network

4. Autorizar nuevos dispositivos:

  • Ve a https://my.zerotier.com/ > tu red
  • En "Members" verás nuevos dispositivos sin autorizar
  • Haz clic en el cuadrado azul para autorizar cada uno

5. Modificar Servidor.java para escuchar en ZeroTier:

Cambiar la línea del ServerSocket:

// De: ServerSocket serverSocket = new ServerSocket(PUERTO); // A: ServerSocket serverSocket = new ServerSocket(PUERTO, 50, InetAddress.getByName("0.0.0.0"));

6. Modificar Cliente.java con IP ZeroTier del servidor:

Cambiar esta línea:

private static final String HOST = "localhost"; // Por la IP ZeroTier del servidor: private static final String HOST = "192.168.196.42";

7. Recompilar y ejecutar
